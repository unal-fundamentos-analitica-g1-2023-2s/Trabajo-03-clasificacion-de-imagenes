3.      Modelo Convolucional (CNN) - Arquitectura y Parmetros

Las redes neuronales convolucionales (CNN) representan un enfoque poderoso para la clasificacin de imgenes, especialmente en el contexto de nuestro proyecto. En esta seccin, examinaremos detalladamente la arquitectura de un modelo CNN, las diferentes capas que lo componen, as como las elecciones cruciales en trminos de hiperparmetros, optimizadores, funciones de prdida y funciones de activacin.

A)      Arquitectura de un Modelo CNN

 

Un modelo CNN tpico consta de capas convolucionales, capas de pooling y capas completamente conectadas. Las capas convolucionales detectan patrones en la imagen, las capas de pooling reducen las dimensiones, y las capas completamente conectadas realizan la clasificacin.

 

B)      Capas del Modelo

 

- Capas Convolucionales: Estas capas aplican filtros para detectar caractersticas especficas en la imagen, permitiendo una jerarqua de representaciones.

- Capas de Pooling: Estas capas reducen las dimensiones espaciales de los mapas de caractersticas, preservando la informacin esencial.

- Capas Completamente Conectadas: Estas capas realizan la clasificacin final integrando las caractersticas extradas.

 

C)      Hiperparmetros a Elegir:

 

- Tamao del Ncleo Convolucional: Determina el tamao del filtro aplicado durante la convolucin.

- Stride: Representa el nmero de pxeles que se desplazan el filtro durante la convolucin.

- Nmero de Filtros: Indica la cantidad de filtros utilizados para extraer diferentes caractersticas.

- Tamao de la Ventana de Pooling: Determina el tamao de la regin para la reduccin dimensional.

 

D)     Optimizadores:

 

- Adam, SGD, RMSprop: Estos optimizadores ajustan los pesos del modelo para minimizar la funcin de prdida durante el entrenamiento.

 

E)      Funciones de Prdida:

 

- Entropa Cruzada Binaria (Binary Crossentropy): Utilizada para problemas de clasificacin binaria.

- Entropa Cruzada Categrica (Categorical Crossentropy): Adecuada para problemas de clasificacin multiclase.

 

F)       Funciones de Activacin:

 

- ReLU (Rectified Linear Unit): Comnmente utilizada para la activacin de las capas convolucionales.

- Sigmoid o Softmax: Empleadas para la capa de salida, segn la naturaleza del problema (binario o multiclase).

El objetivo principal de la ampliacin de datos es mejorar la generalizacin del modelo al exponerlo a una mayor variedad de escenarios y puntos de vista, simulando as condiciones del mundo real. Esto permite que el modelo aprenda caractersticas ms robustas y aumente su capacidad para reconocer patrones relevantes en diferentes situaciones.

Las transformaciones comnmente aplicadas durante la ampliacin de datos incluyen rotacin, volteo horizontal o vertical, zoom, desplazamiento y otras modificaciones similares. Estas transformaciones se aplican de manera aleatoria a cada lote de entrenamiento, proporcionando variedad adicional al modelo mientras se mantienen inalteradas las etiquetas de clase.

En resumen, la ampliacin de datos ayuda a mitigar el sobreajuste del modelo a datos de entrenamiento especficos, mejorando as su capacidad para generalizar y rendir mejor en nuevos datos.
